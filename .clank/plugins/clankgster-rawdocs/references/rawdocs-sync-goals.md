# rawdocs sync goals

Canonical priority goals for rawdocs structural sync behavior.

## Priority order

1. Preserve author meaning and style from rawdocs.
2. Apply subtractive sync by default: when rawdocs no longer supports a topic, remove corresponding non-`rawdocs/` outputs unless an explicit exception is recorded.
3. Respect continuity from existing structure for traceable paths.
4. Evolve structure when rawdocs growth or clarity demands it.
5. Never alter `rawdocs/`.

## Operational interpretation

- Source truth is what rawdocs currently supports, not what prior output happened to contain.
- Continuity is guidance, not a veto against subtractive sync.
- Churn should be minimized for retained traceable paths, but not by preserving orphaned files.
